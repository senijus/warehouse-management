# 轻量级仓库库存管理系统 - 项目规划文档

## 一、项目概述

针对仓储环境中设备信息分散、操作不可追溯与权限薄弱问题，设计并实现基于嵌入式 Linux 的库存管理系统。结合 Web 服务与本地数据库，实现数据持久化、操作日志记录与可视化管理。

### 技术栈
- **语言**: C语言
- **数据库**: SQLite3
- **并发模型**: I/O 多路复用（epoll，单线程事件驱动）
- **网络协议**: Socket 网络编程 + HTTP/1.1
- **前端**: 原生 HTML/CSS/JavaScript

```
┌─────────────┐     HTTP/1.1      ┌──────────────────────────────────┐
│   浏览器     │ ◄──────────────► │         epoll 并发服务器          │
│  (多终端)    │                   │                                  │

                                  │  │ HTTP解析  │  │   路由分发    │  │
                                  │  │ 请求/响应 │→│  handler.c   │  │
                                  │  └──────────┘  └──────┬───────┘  │
                                  │                       │          │
                                  │  ┌──────────┐  ┌──────▼───────┐  │
                                  │  │ auth.c   │  │ database.c   │  │
                                  │  │ 认证鉴权  │  │  SQLite操作  │  │
                                  │  └──────────┘  └──────────────┘  │
                                  └──────────────────────────────────┘
                                               │
                                        ┌──────▼──────┐
                                        │ inventory.db │
                                        │  SQLite数据库 │
                                        └─────────────┘
```

### 核心数据流
```
客户端请求 → epoll_wait 接收 → HTTP 报文解析 → 路由匹配
  → 认证检查 → 权限校验 → 业务处理 → SQLite 读写 + 操作日志
  → 动态生成 HTTP 响应 → 发送回客户端
```

## 四、模块设计

### 4.1 服务器模块 (server.c/h)

负责 TCP 连接管理和 epoll 事件循环。

```c
// 核心接口
int  server_init(int port);           // 创建 socket、绑定、监听、初始化 epoll
void server_run(void);                // 主事件循环
void server_add_client(int fd);       // 将新客户端加入 epoll
void server_remove_client(int fd);    // 移除客户端连接
```

**设计要点:**
- 非阻塞 socket + epoll 边沿触发 (EPOLLET)
- 最大并发连接数: 1024
- 读缓冲区: 每连接 8KB
- 超时检测: 空闲连接 60s 自动关闭

### 4.2 HTTP 解析模块 (http.c/h)

解析 HTTP/1.1 请求报文，生成响应报文。

```c
// 请求结构体

// 核心接口
int  http_parse_request(const char *raw, int len, HttpRequest *req);
void http_send_response(int fd, int status, const char *content_type, const char *body, int body_len);

void http_parse_form_body(const char *body, const char *key, char *value, int value_size);
```

### 4.4 认证模块 (auth.c/h)

基于 Cookie 的会话认证和角色权限管理。

```c
// 核心接口
int  auth_login(HttpRequest *req, int fd);            // 处理登录请求
int  auth_check(HttpRequest *req);                    // 检查是否已登录
int  auth_require_role(HttpRequest *req, const char *role);  // 角色校验
void auth_get_username(HttpRequest *req, char *username, int size);  // 获取当前用户名
void auth_logout(int fd);                             // 清除 session
```

**认证流程:**
1. 用户提交用户名密码 → `POST /login`
2. 查询数据库验证 → 成功则 `Set-Cookie: session=<username>`
3. 后续请求从 Cookie 提取用户名 → 查询数据库验证用户存在
4. 管理员接口额外校验 `role == 'admin'`

## 五、并发模型

```
                    ┌─────────────────────┐
                    │    main 主循环       │
                    │    epoll_wait()      │
                    └──────────┬──────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
        新连接事件         可读事件          可写事件
        accept()          read()           write()
              │                │                │
              ▼                ▼                ▼
        加入 epoll      HTTP 解析+处理    发送响应
        设置非阻塞      路由→业务→数据库   关闭或保持
```

**单线程 epoll 优势:**
- 无锁、无线程切换开销
- 内存占用最小
- 适合嵌入式低资源环境
- 代码逻辑清晰，调试简单

## 六、构建与运行

### Makefile
```makefile
CC = gcc
CFLAGS = -Wall -Wextra -O2
LDFLAGS = -lsqlite3
TARGET = server

all: $(TARGET)

$(TARGET): main.c server.c http.c handler.c database.c auth.c
	$(CC) $(CFLAGS) -o $@ $^ $(LDFLAGS)

clean:
	rm -f $(TARGET)
```

### 运行
```bash
make
./server          # 默认监听 8080 端口
./server 9090     # 指定端口
```

### 访问
```
http://localhost:8080          # 登录页
默认管理员: admin / admin123
```

## 七、功能特性

1. **用户管理**: 管理员可添加/删除用户，区分 admin 和 operator 角色
2. **设备管理**: 添加/删除设备信息（名称、类别、位置、描述）
3. **库存操作**: 入库增加库存、出库减少库存，自动校验库存余量
4. **操作日志**: 所有操作自动记录（谁、什么操作、什么设备、多少数量、何时）
5. **权限控制**: 操作员只能入库/出库/查询，管理员可管理设备和用户
6. **并发访问**: epoll 多终端同时访问，无阻塞
7. **页面重定向**: 操作成功后自动跳转，登录/登出流程顺畅
